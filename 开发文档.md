## 第一章 项目概述
### 1.1 项目目标
打造面向大学生的**零配置 MySQL 在线学习实训平台**，内置固定练习数据库与课程同步学习大纲，学生边学知识点边实时执行 SQL，自动记录学习进度与错题，解决高校数据库课程实验环境配置难、理论实操脱节的痛点。

### 1.2 核心功能范围（MVP 必做）
| 模块 | 功能点 | 优先级 |
|------|--------|--------|
| 用户认证 | 学生注册、登录、JWT 鉴权、退出登录 | P0 |
| 学习大纲 | 分级章节+任务树展示、闯关解锁、学习状态标记 | P0 |
| 任务详情 | 知识点讲解、课堂例题、课后练习展示 | P0 |
| SQL 编辑器 | MySQL 语法高亮、格式化、快捷键执行、例题一键填充 | P0 |
| SQL 执行 | 固定练习库执行、结果表格渲染、错误提示 | P0 |
| 学习进度 | 任务标记已学习/未学习、总进度统计、章节完成率 | P0 |
| 错题本 | 执行失败自动收录、错题列表、标记已掌握、一键重做 | P1 |
| 数据重置 | 练习库一键恢复初始数据 | P1 |
| 查询历史 | 执行记录保存、列表展示、一键复用 | P1 |

### 1.3 非功能要求
- 响应速度：SQL 执行接口平均响应 < 200ms，页面首屏加载 < 2s
- 并发支持：单实例支持 200 学生同时在线使用
- 安全要求：禁止高危 SQL 执行，学生数据严格隔离
- 兼容性：支持 Chrome、Edge 主流浏览器，适配 1080P 及以上分辨率

## 第三章 数据库设计
### 3.1 数据库规划
共 2 个数据库，物理隔离，权限分离：
1.  **系统库**：`mysql_study_system`，存储业务数据，使用 root 账号读写
2.  **练习库**：`mysql_study_practice`，学生练习用，仅授予 `SELECT/INSERT/UPDATE/DELETE` 权限

### 3.2 系统库完整表设计

### 3.3 系统库完整建表SQL
```sql
CREATE DATABASE IF NOT EXISTS mysql_study_system DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;
USE mysql_study_system;

-- 1. 学生用户表
CREATE TABLE sys_student (
    id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '学生主键ID',
    username VARCHAR(50) NOT NULL UNIQUE COMMENT '登录账号/学号',
    password VARCHAR(255) NOT NULL COMMENT 'BCrypt加密密码',
    real_name VARCHAR(50) COMMENT '真实姓名',
    college VARCHAR(100) COMMENT '所属院系',
    major VARCHAR(100) COMMENT '所属专业',
    status TINYINT DEFAULT 1 COMMENT '状态：1正常 0禁用',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='学生用户表';

-- 2. 学习章节表
CREATE TABLE study_chapter (
    id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '章节ID',
    chapter_name VARCHAR(100) NOT NULL COMMENT '章节名称',
    sort_order INT NOT NULL DEFAULT 0 COMMENT '排序号',
    description VARCHAR(255) COMMENT '章节简介',
    pre_chapter_id BIGINT DEFAULT 0 COMMENT '前置解锁章节ID',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='学习章节表';

-- 3. 学习任务表
CREATE TABLE study_task (
    id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '任务ID',
    chapter_id BIGINT NOT NULL COMMENT '所属章节ID',
    task_name VARCHAR(100) NOT NULL COMMENT '任务名称',
    sort_order INT NOT NULL DEFAULT 0 COMMENT '章节内排序号',
    knowledge_content TEXT COMMENT '知识点讲解',
    example_sql TEXT COMMENT '课堂例题SQL',
    practice_question TEXT COMMENT '课后练习题要求',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    INDEX idx_chapter_id(chapter_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='学习任务表';

-- 4. 学生学习进度表
CREATE TABLE student_task_progress (
    id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '主键ID',
    student_id BIGINT NOT NULL COMMENT '学生ID',
    task_id BIGINT NOT NULL COMMENT '任务ID',
    study_status TINYINT NOT NULL DEFAULT 0 COMMENT '0未学习 1已学习',
    study_time DATETIME COMMENT '完成学习时间',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    UNIQUE KEY uk_stu_task(student_id, task_id),
    INDEX idx_student_id(student_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='学生学习进度表';

-- 5. 学生错题本表
CREATE TABLE student_error_book (
    id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '主键ID',
    student_id BIGINT NOT NULL COMMENT '学生ID',
    sql_content TEXT NOT NULL COMMENT '错误SQL内容',
    error_msg TEXT NOT NULL COMMENT '错误信息',
    master_status TINYINT DEFAULT 0 COMMENT '0未掌握 1已掌握',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    INDEX idx_student_id(student_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='学生错题本表';

-- 6. 查询历史表
CREATE TABLE query_history (
    id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '主键ID',
    student_id BIGINT NOT NULL COMMENT '学生ID',
    sql_content TEXT NOT NULL COMMENT 'SQL内容',
    execute_status TINYINT NOT NULL COMMENT '1成功 0失败',
    cost_time INT COMMENT '执行耗时(ms)',
    error_msg TEXT COMMENT '错误信息',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '执行时间',
    INDEX idx_stu_time(student_id, create_time)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='查询历史表';
```

### 3.4 练习库设计
#### 3.4.1 练习库表结构
采用高校教材经典「院系-学生-课程-成绩」模型：
```sql
CREATE DATABASE IF NOT EXISTS mysql_study_practice DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;
USE mysql_study_practice;

-- 院系表
CREATE TABLE department (
    dept_id INT PRIMARY KEY COMMENT '院系编号',
    dept_name VARCHAR(50) NOT NULL COMMENT '院系名称',
    dept_head VARCHAR(50) COMMENT '系主任',
    location VARCHAR(50) COMMENT '所在校区'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='院系表';

-- 学生表
CREATE TABLE student (
    stu_id INT PRIMARY KEY COMMENT '学号',
    stu_name VARCHAR(50) NOT NULL COMMENT '姓名',
    gender CHAR(2) COMMENT '性别',
    birthday DATE COMMENT '出生日期',
    dept_id INT COMMENT '所属院系编号',
    admission_date DATE COMMENT '入学日期',
    INDEX idx_dept(dept_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='学生表';

-- 课程表
CREATE TABLE course (
    course_id INT PRIMARY KEY COMMENT '课程编号',
    course_name VARCHAR(100) NOT NULL COMMENT '课程名称',
    credit DECIMAL(3,1) COMMENT '学分',
    teacher VARCHAR(50) COMMENT '授课教师',
    dept_id INT COMMENT '开课院系'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='课程表';

-- 成绩表
CREATE TABLE score (
    id INT PRIMARY KEY AUTO_INCREMENT,
    stu_id INT NOT NULL COMMENT '学号',
    course_id INT NOT NULL COMMENT '课程编号',
    score DECIMAL(5,1) COMMENT '考试成绩',
    exam_date DATE COMMENT '考试日期',
    UNIQUE KEY uk_stu_course(stu_id, course_id),
    INDEX idx_stu(stu_id),
    INDEX idx_course(course_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='成绩表';
```

#### 4.4.3 JWT工具类 JwtUtil.java
```java
package com.mysql.study.common;

import io.jsonwebtoken.Claims;
import io.jsonwebtoken.Jwts;
import io.jsonwebtoken.SignatureAlgorithm;
import org.springframework.beans.factory.annotation.Value;
import org.springframework.stereotype.Component;

import java.util.Date;

@Component
public class JwtUtil {

    @Value("${jwt.secret}")
    private String secret;

    @Value("${jwt.expire-hours}")
    private Long expireHours;

    public String generateToken(Long userId, String username) {
        Date now = new Date();
        Date expireDate = new Date(now.getTime() + expireHours * 3600 * 1000);
        return Jwts.builder()
                .setSubject(String.valueOf(userId))
                .claim("username", username)
                .setIssuedAt(now)
                .setExpiration(expireDate)
                .signWith(SignatureAlgorithm.HS256, secret)
                .compact();
    }

    public Claims parseToken(String token) {
        try {
            return Jwts.parser()
                    .setSigningKey(secret)
                    .parseClaimsJws(token)
                    .getBody();
        } catch (Exception e) {
            return null;
        }
    }

    public Long getUserId(String token) {
        Claims claims = parseToken(token);
        return claims != null ? Long.valueOf(claims.getSubject()) : null;
    }
}
```

#### 4.5.2 SQL执行引擎模块

```ts
import axios from 'axios'
import { message } from 'antd'
import { useUserStore } from '@/store/userStore'

const request = axios.create({
  baseURL: '/api',
  timeout: 30000
})

// 请求拦截器
request.interceptors.request.use(
  (config) => {
    const token = useUserStore.getState().token
    if (token) {
      config.headers.Authorization = `Bearer ${token}`
    }
    return config
  },
  (error) => Promise.reject(error)
)

// 响应拦截器
request.interceptors.response.use(
  (response) => {
    const res = response.data
    if (res.code === 200) {
      return res.data
    } else if (res.code === 401) {
      message.error('登录已过期，请重新登录')
      useUserStore.getState().logout()
      window.location.href = '/login'
      return Promise.reject(res.message)
    } else {
      message.error(res.message)
      return Promise.reject(res.message)
    }
  },
  (error) => {
    message.error(error.message || '网络请求失败')
    return Promise.reject(error)
  }
)
```

## 第六章 前后端联调规范
### 6.1 统一响应格式
所有接口必须遵循以下格式，前端统一拦截处理：
```json
{
  "code": 200,
  "message": "操作成功",
  "data": {}
}
```
- `code=200` 表示成功，前端取 `data` 字段
- `code≠200` 表示失败，前端弹出 `message` 提示

### 6.2 鉴权规范
1.  登录成功后，后端返回 `token` 字段
2.  前端将 token 存入 localStorage，后续所有请求头携带 `Authorization: Bearer {token}`
3.  token 过期时，后端返回 `code=401`，前端清空本地数据并跳转登录页
4.  后端从 token 中解析学生ID，禁止前端通过参数传递学生ID

### 6.3 分页规范
- 分页请求参数统一使用 `pageNum`、`pageSize`
- 分页响应统一格式：
```json
{
  "total": 100,
  "list": [],
  "pageNum": 1,
  "pageSize": 20
}
```

### 6.4 跨域配置
后端配置全局跨域，允许前端开发地址 `http://localhost:3000` 访问，生产环境通过 Nginx 反向代理解决跨域。

### 6.5 错误码约定
| 错误码 | 含义 | 前端处理 |
|--------|------|----------|
| 200 | 成功 | 正常处理数据 |
| 400 | 参数错误 | 弹出错误提示 |
| 401 | 未登录/登录过期 | 跳转登录页 |
| 403 | 无权限 | 弹出无权限提示 |
| 500 | 系统异常 | 弹出系统异常提示 |
| 1001 | 高危SQL禁止执行 | 弹出危险操作提示 |
| 1002 | SQL执行失败 | 弹出具体错误信息 |

## 第七章 开发规范
### 7.1 后端开发规范
1.  **命名规范**：类名大驼峰，方法名/变量名小驼峰，常量全大写下划线分隔，数据库表名下划线分隔
2.  **分层规范**：Controller 只做参数校验和返回，业务逻辑写在 Service，数据库操作写在 Mapper
3.  **异常处理**：业务异常抛出 `BusinessException`，系统异常由全局异常处理器统一捕获
4.  **日志规范**：关键入口打印 info 日志，异常打印 error 日志并附堆栈
5.  **注释规范**：公共方法、复杂逻辑必须加注释，说明功能、参数、返回值

### 7.2 前端开发规范
1.  **命名规范**：组件名大驼峰，文件与组件名一致，变量/函数小驼峰
2.  **TypeScript 规范**：严格模式，禁止滥用 `any`，所有接口数据定义类型
3.  **组件规范**：公共组件抽离，Props 明确定义类型，职责单一
4.  **样式规范**：使用 CSS Modules 或 antd 主题定制，避免全局样式污染
5.  **接口规范**：所有接口在 `api` 目录统一管理，页面不直接调用 axios

### 7.3 Git 提交规范
提交信息格式：`type: 提交内容描述`
- `feat`：新增功能
- `fix`：修复bug
- `docs`：文档更新
- `style`：代码格式调整
- `refactor`：代码重构
- `perf`：性能优化
- `chore`：构建/工具配置更新
